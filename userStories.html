User Stories

1.1: Animations

Intro: As a player I want to see smooth animations in the game to make it visually appealing.

Elaboration: Animations will be a large part of the fighting aspect of the game. They must look nice and semi-realistic.

Constraints: none

Effort Estimation: 20 person hours

Acceptance Test: Create sprites for a specific task and code specific task to showcase animation.

1.2: Tutorial

Intro: As a player, I want to be offered an tutorial page and a move-list so that I learn how to play.

Elaboration: Each game has individual controls and commands. A tutorial will be useful to help users navigate the game mechanics.

Constraints: none

Effort Estimation: 2 person hours

Acceptance Test: Create "tutorial" button next to "play" button (on the main menu) to offer more usability than an in-game walkthrough.

1.3: Balance

Intro: As a player, I want to see the fighting types and stat distribution balanced against each other so that the gameplay is fair.

Elaboration: The game will have different types of fighters. They should all be equally powerful in their own way so that the matches rely on users' strategies.

Constraints: none

Effort Estimation: 8 person hours

Acceptance Test: Let players demo the game and test that they are all generally balanced against each type.

1.4 Items

Intro: As a player, I expect items to work as advertised so that I can use them to my advantage.

Elaboration: Each item has a different purpose. If I decide to use one of the items, I expect it to do what it's supposed to.

Constraints: none

Effort Estimation: 20 person hours

Acceptance Test: Players will try each item in a fight to ensure it works as intended.

1.5 Non-Playable Characters

Intro: As a player, I want the NPCs to fight me in a logical way so that the game is challenging, yet doable.

Elaboration: The NPCs should not be overpowered compared to a player, but their moves should make sense and pose somewhat of a threat to the user. They will have to be programmed with an AI.

Constraints: none

Effort Estimation: 15 person hours

Acceptance Test: Players will demo battles with NPCs and determine if their algorithm is accurate.

1.6 Navigation

Intro: As a player, I expect to have some kind of lobby or cutscene so that I can transition between different battles.

Elaboration: Players will fight a variety of enemies. There must be logical way to move them between enemies so that the story line runs smoothly.

Constraints: none

Effort Estimation: 15 person hours

Acceptance Test: A player will demo moving between fights and decide if the transition makes sense and goes smoothly.
