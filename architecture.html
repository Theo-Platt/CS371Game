Project Overview

This project will be a turn-based combat system in which players can fight Non-Playable Characters. It will include a variety of players stats (such as health, speed, evasiveness, and accuracy) as well as different moves that can heal the player, attack the opponent, or manipulate stats. It will have a simple map, which leads the player into our pokemon-style combat encounters.

Key Architecture Drivers

We need our system to:
1. Have a turn-based battle system
2. Have a map/world to transition between battles
All other requirements fall under these two categories, so we decided to base our architecture designs off these key points.

Architecture Style Choices

For the battle system, we will implement a factory architecture style. We will create a factory function that will generate an enemy object with unique type/stats, then return a reference to that object. We also plan to use function overloading so that our factory can work with a variety of parameters (i.e., if we want the enemy to have a set type). We considered using a cloning architecture to create new enemies from existing prototypes, but we decided that the factory style would allow for more flexibility for creating enemies.
For our map, we will use an observer architecture style. When the user presses a button to move their character or start a battle, an observer will notify the appropriate objects of the change in state, which will trigger other actions (such as walking animations or a transition into the battle screen). We decided that this would be more efficient than a notification system, which would spend too much time polling.
